## Resolution Framework

When a disagreement surfaces, follow this consent-based process:

1. **Classify** the disagreement: Factual / Strategic / Aesthetic / Values
2. **Factual** → resolve with A/B test or customer data (no opinion needed)
3. **Aesthetic** → domain owner decides:
   - **Andrea** = sensory experience, product language, ingredient communication
   - **Camila** = philosophy, structure, conceptual framing
4. **Strategic / Values** → consent process:
   - One founder writes a **proposal**
   - Other founder asks **clarifying questions**
   - Check for **OBJECTIONS** (concrete risk or harm to the brand) vs **CONCERNS** (personal preferences — don't block)
   - If no objections: accept as **"good enough for now, safe enough to try"**
   - Set a **90-day review date**
5. **Log** every resolution in `DECISIONS.md` with rationale and date
